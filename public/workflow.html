How the CI/CD Workflow Runs

This page shows how code moves from a GitHub push into a deployed Render release.

1

Developer pushes code to GitHub

The workflow starts when code is pushed to the main branch or a pull request targets main.

2

CI validates the code

GitHub Actions installs dependencies, runs linting, runs Jest tests, and checks the build step. If any check fails, deployment stops here.

3

Docker image is created

When CI passes, the pipeline builds a Docker image from the Dockerfile so the app runs in a repeatable environment.

4

Image is pushed and deploy hook runs

The image is tagged and pushed to Docker Hub, then GitHub Actions calls the Render deploy hook to start production deployment.

Where to Look in the Project

  • CI/CD workflow: .github/workflows/deploy.yml
  • Container build: Dockerfile
  • App server: index.js
  • Live run history: Pipeline Dashboard

What CI/CD Is For

CI

Find problems early

Tests and linting catch broken changes before they reach production.

CD

Deploy automatically

Once checks pass, the exact same code gets packaged and deployed without manual steps.

The live deployment is visible in the dashboard because it reads the latest GitHub Actions run and the deployed commit SHA.
